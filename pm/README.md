# pm — 五层产品分析方法论技能集

pm 是一套结构化的产品分析方法论技能集合，从第一性原理的业务目的出发，逐层推导：业务目的 → 边界与终端 → 用户画像 → 场景旅程 → 场景功能规格，并配套单层检查、跨层验证、认知捕获、用户模拟与测试用例生成，形成"分析—验证—落地"的完整闭环。

## 五层主链

| 层级 | 技能 | 职责 |
|------|------|------|
| L1 | pm-layer1 | 业务目的层 — 从第一性原理明确"为什么做" |
| L2 | pm-layer2 | 目的边界层 — 识别主体、终端、划分边界 |
| L3 | pm-layer3 | 边界用户层 — 推导用户画像、痛点、使用环境 |
| L4 | pm-layer4 | 用户场景层 — 画旅程、标注接触点、编写场景故事 |
| L5 | pm-layer5 | 场景功能层 — 抽取能力、设计功能架构、产出规格 |

## 辅助技能

- **pm-init** — 项目初始化：采集原始想法，创建项目骨架和 CLAUDE.md
- **pm-capture** — 产品认知捕获：从对话中提取并持久化决策、洞察、框架认知
- **pm-check-\*** — 单层检查（l1~L5）与跨层验证（l1-l2 / l2-l3 / l3-l4 / l4-l5），校验层级间约束传递与一致性
- **pm-simulate** — 小白用户模拟：角色扮演 sub-agent 发现设计不合理之处
- **pm-test** — 测试用例生成：按用户×终端串通功能旅程，发现断链和缺口

> 技能清单可由 `publish.sh list` 自动列出。
