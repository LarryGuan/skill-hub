# L4 产出格式模板

## 文件一：04-用户场景/{终端名}/scenario-{关键词}.md

### 场景故事标准格式

```markdown
# {场景名称}

## 基本信息
- **关系事件**：{L2 事件序号 + 描述}
- **终端**：{终端名}
- **触发条件**：{Situation —— 什么情况下启动}
- **参与者**：{L3 画像主体列表，标注谁是发起者/响应者/受影响者}
- **场景分类**：核心 / 边缘 / 异常

## 前置条件
{场景开始前必须满足的条件，如"用户已登录"、"商品已加入购物车"}

## 用户旅程
| 步骤 | 用户操作 | 系统响应 | 痛点命中 |
|------|---------|---------|---------|
| 1 | {操作描述} | 🔴 {系统做什么} | {L3 痛点引用，无则留空} |
| 2 | {操作描述} | ⚪ 用户独立完成 | |
| 3 | {操作描述} | 🔴 {系统做什么} | {L3 痛点引用} |
| ... | ... | ... | ... |

## 决策点
{旅程中的关键判断节点，如"库存不足？→ 走补货流程"}

## 设备环境约束
{从 L3 使用环境引用本终端的约束（网络/时间/干扰/能力）}

## 后置条件
{场景结束后系统状态}

## 业务意图
{这个场景说明了什么业务价值？对齐了什么理解？}

## 活动图
```mermaid
flowchart TD
    ...
```
```

### 命名规范

| 场景类型 | 命名格式 | 示例 |
|---------|---------|------|
| 核心场景 | `scenario-{事件关键词}.md` | `scenario-浏览商品.md` |
| 异常场景 | `scenario-{事件关键词}-{异常描述}.md` | `scenario-下单-库存不足.md` |
| 边缘场景 | `scenario-{事件关键词}.md` | `scenario-收藏商品.md` |

### 字段说明

| 字段 | 要求 | 常见错误 |
|------|------|----------|
| 基本信息 | 5 个要素齐全（事件、终端、触发条件、参与者、分类） | 遗漏场景分类或触发条件 |
| 前置条件 | 具体可判断 | 写成功能描述 |
| 用户旅程 | 步骤有编号，系统响应标注 🔴/⚪ | 遗漏系统接触点标注 |
| 决策点 | 标注判断条件和分支走向 | 只写正常路径 |
| 设备环境 | 从 L3 引用，不编造 | 与 L3 使用环境矛盾 |
| 后置条件 | 描述系统状态变化 | 遗漏 |
| 业务意图 | 一句话说明对齐价值 | 写成功能说明 |
| 活动图 | 泳道=参与者，含决策点 | 画成系统流程图 |

## 文件二：04-用户场景/index.md

### 标准格式

```markdown
# 用户场景

## 约束摘要（供下层快速引用）
- 核心场景：[2-3 个最关键场景的一句话概括]
- 终端分布：[哪些终端，各有多少场景]
- 强制回溯：场景故事列表 + 交叉分析（L5 必须全文回溯）

## 场景总览
| 终端 | 核心场景 | 边缘场景 | 异常场景 | 关系事件覆盖 |
|------|---------|---------|---------|-------------|
| {终端A} | {n} | {n} | {n} | {事件列表} |
| {终端B} | {n} | {n} | {n} | {事件列表} |

## 排除项（不依赖系统的事件）
| 事件 | 排除原因 |
|------|---------|
| {序号+描述} | {原因} |

## 交叉分析
### 依赖关系
{场景间的先后依赖，如"A 的后置条件是 B 的前置条件"}

### 冲突检测
{同一数据/资源在不同场景中的操作冲突}

### 数据流
{场景间的数据传递路径}

## 用例图细化版
```mermaid
...
```
```

### 字段说明

| 字段 | 要求 | 常见错误 |
|------|------|----------|
| 约束摘要 | 200 字以内 | 太详细或遗漏关键场景 |
| 场景总览 | 每个终端一行，数字统计 | 终端遗漏 |
| 排除项 | 每个排除事件有原因 | 排除无理由 |
| 交叉分析 | 三维度齐全（依赖/冲突/数据流） | 只做依赖，忽略冲突 |
| 用例图细化版 | 覆盖所有场景 | 与 L2 初版断裂 |

## 文件三：04-用户场景/CLAUDE.md

### 标准格式

```markdown
# L4 约束摘要
{200字以内：场景总览、终端场景分布要点}

## ⚠️ 强制回溯项（下游必须全文读取，不能只读摘要）
- 场景故事列表（L5 每个功能必须覆盖至少一个场景）
- 交叉分析（L5 必须验证终端间数据一致性）

## 下游须知
- 场景交叉分析位置：04-用户场景/index.md 的交叉分析章节
- 每个终端的场景故事位置：04-用户场景/{终端名}/
- 场景源自 L2 业务模型中的关系事件，下游功能必须覆盖所有场景
- 用例图细化版在 index.md 中，L5 以此为基础
- 每个场景含业务意图，L5 功能规格应回溯到对应场景
```

### 编写规则

| 字段 | 规则 |
|------|------|
| 约束摘要 | 从 index.md 的约束摘要章节提取，200 字以内 |
| 强制回溯项 | 固定包含：场景故事列表 + 交叉分析 |
| 下游须知 | 列出 L5 必须注意的场景位置、用例图位置、业务意图回溯要求 |
