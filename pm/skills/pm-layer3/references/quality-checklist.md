# L3 质量检查清单

## 完整性检查

- [ ] 所有 L2 主体都有对应画像？（不能遗漏间接主体）
- [ ] 每个画像 6 个要素齐全？（角色定位、典型特征、核心目标、动机、行为模式、心理特征）
- [ ] 痛点列表中每个痛点都关联到 L2 业务事件？
- [ ] 痛点列表覆盖所有主体？（每个主体至少 1 个核心痛点）
- [ ] 使用环境描述覆盖所有主体？
- [ ] 使用环境与 L2 终端分工表一致？（终端类型对得上）
- [ ] 产品上下文包含 L1+L2+L3 三个维度？

## 质量检查

- [ ] 画像特征是否影响设计？（无关特征如"喜欢打篮球"不应出现）
- [ ] 痛点是否具体可感知？（不是"体验不好""效率低"）
- [ ] 痛点严重程度判断是否合理？（不全是"核心"，有层次区分）
- [ ] 使用环境是否描述了约束？（不只是"用手机"，要描述信号/时间/干扰）
- [ ] 产品上下文 L1 部分是否与 01-业务目的/index.md 一致？
- [ ] 产品上下文 L2 部分是否与 02-目的边界/index.md 一致？
- [ ] 产品上下文 L3 是否只增加了"为谁做"维度？（没有新增 L1/L2 范围的内容）

## 常见错误自查

- [ ] 是否避免了画像太详细变成用户研究报告？
- [ ] 是否避免了痛点脱离 L2 业务事件？
- [ ] 是否避免了环境描述与终端分工矛盾？
- [ ] 是否避免了遗漏 L2 中的间接主体？
- [ ] 是否避免了产品上下文增加不属于 L3 的维度？
- [ ] 是否避免了所有痛点都标"核心"？（应有次要痛点）

## 框架检查

- [ ] 03-边界用户/CLAUDE.md 已生成？（约束摘要 + 强制回溯项 + 下游须知）
- [ ] 03-边界用户/CLAUDE.md 的约束摘要控制在 200 字以内？
- [ ] 03-边界用户/CLAUDE.md 的强制回溯项包含痛点列表？
- [ ] 项目 index.md 的 L3 进度标记已更新为 [x]？
- [ ] 项目根 CLAUDE.md 未被修改？（框架宪法，只读）

## 写入前
