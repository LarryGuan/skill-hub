# L5 产出格式模板

## 文件零：05-场景功能/plan.md — 执行计划+进度追踪

### 标准格式

```markdown
# pm-L5 执行计划

## 状态：进行中 / 已完成

## 拆分结果

- 阶段一：领域模型（跨终端共享，一次完成）
- 阶段二：共 N 个子单元（按终端拆分）
  - 子单元 A（{终端名}）：负责场景 {列表}
  - 子单元 B（{终端名}）：负责场景 {列表}
- 阶段三：共 N 个子单元（按终端拆分）

### 排除项（如有）
| 场景 | 排除原因 |
|------|---------|

### 上下文打包
- 阶段一需读取：L4 index.md + L2 类图初版 + 所有终端的 scenario
- 阶段二每终端需读取：领域模型 + 本终端的 L4 scenario 文件
- 阶段三每终端需读取：本终端的页面规格 + 领域模型

### 依赖关系
- 阶段二依赖阶段一（领域模型必须先完成）
- 阶段三依赖阶段二（页面规格必须先完成）
- 各终端间无依赖（可并行）

## 进度追踪

### 阶段一：领域模型
| 子步骤 | 状态 | 产出文件 | 备注 |
|--------|------|---------|------|
| 类图细化 | [ ] | models/class.md | |
| 状态机图 | [ ] | models/states/{实体名}.md | |
| 规则矩阵 | [ ] | models/rules/{领域}-rules.md | |
| 检查点 B | [ ] | | |

### 阶段二：终端交付
| 终端 | 功能架构 | 页面流程 | 页面规格 | 页面原型 | 检查点C | 状态 |
|------|---------|---------|---------|---------|--------|------|
| {终端A} | [ ] | [ ] | [ ] | [ ] | [ ] | 未开始 |
| {终端B} | [ ] | [ ] | [ ] | [ ] | [ ] | 未开始 |

### 阶段三：接口层
| 终端 | API接口视图 | 状态 |
|------|-----------|------|

## 场景覆盖矩阵
| L4 场景 | 终端 | 页面 | API | 规则 | 状态 |
|---------|------|------|-----|------|------|

## 检查点记录

### 检查点 A（Plan 后，执行前）
- 日期：
- 发现：
- 处理：
- 状态：通过 / 需补充

### 检查点 B（领域模型后）
- 日期：
- 覆盖性发现：
- 运行闭环发现：
- 处理：
- 状态：通过 / 需补充

### 检查点 C（每终端后）
#### {终端名}
- 日期：
- 逻辑闭环发现：
- 运行闭环发现：
- 痛点覆盖发现：
- 处理：
- 状态：通过 / 需补充

### 检查点 D（组装前，全局循证）
- 日期：
- 全局覆盖发现：
- 跨终端运行闭环发现：
- 价值闭环发现：
- 处理：
- 状态：通过 / 需补充

## 决策记录

### 决策 1：{标题}
- 触发：
- 选项：
- 决策：
- 原因：
- 影响：

## 文件清单
{随产出更新，列出所有已写入的文件路径}
```

### 命名与位置

| 项目 | 规范 |
|------|------|
| 文件名 | `plan.md` |
| 位置 | `05-场景功能/plan.md` |
| 创建时机 | Plan 阶段用户确认后 |
| 更新时机 | 每个子步骤/检查点完成后 |
| 读取时机 | 断点续传时从磁盘读取恢复进度 |

### 字段说明

| 字段 | 要求 | 常见错误 |
|------|------|----------|
| 状态 | 全局状态标记 | 忘记更新为"已完成" |
| 进度追踪 | 每个子步骤勾选 | 只勾到阶段级，不够细 |
| 场景覆盖矩阵 | 随执行逐步填充 | 一次性填写（应该逐步） |
| 检查点记录 | 每个检查点都有发现和处理 | 只写"通过"不写具体发现 |
| 决策记录 | 重要决策才记录 | 把每个确认都当决策 |
| 文件清单 | 列出所有已写入的文件路径 | 遗漏文件 |

## 文件一：05-场景功能/models/class.md — 类图细化版

| 实体名 | 核心字段 | 来源场景 | 关联实体 |
|--------|---------|---------|---------|

## 实体详细定义

| 字段名 | 类型 | 说明 | 来源场景 |
|--------|------|------|---------|

**关系**：
| 关系类型 | 目标实体 | 说明 |
|----------|---------|------|
| 关联/聚合/组合 | {实体名} | {关系描述} |

**类图**：
```mermaid
classDiagram
    class {实体名} {
        +字段名: 类型

    {实体名} --> {关联实体}

## 文件六：05-场景功能/{终端名}/pages/{页面名}/spec.md — 页面功能规格

### 标准格式

```markdown
# {页面名} 功能规格

## 基本信息
- **终端**：{终端名}
- **对应场景**：{L4 scenario 文件名}
- **承载功能**：{功能编号列表}

## 交互行为
| 元素 | 类型 | 用户操作 | 系统响应 |
|------|------|---------|---------|

## 规则落地
| 规则 ID | 规则名称 | 本页面表现 |
|---------|---------|-----------|
| R-001 | {名称} | {用户感知表现} |

## 数据绑定
| 展示项 | 绑定实体 | 绑定字段 | 来源 |
|--------|---------|---------|------|
| {展示内容} | {实体名} | {字段名} | GET {接口路径} |

## 状态影响
| 操作 | 触发状态转换 | 引用状态机 |
|------|-------------|-----------|
| {用户操作} | {状态A} → {状态B} | {实体名}.md |

## 场景回溯
- 本页面服务于场景：{场景名}（{scenario 文件名}）
```

### 命名与位置

| 项目 | 规范 |
|------|------|
| 文件名 | `spec.md` |
| 位置 | `05-场景功能/{终端名}/pages/{页面名}/spec.md` |

### 字段说明

| 字段 | 要求 | 常见错误 |
|------|------|----------|
| 交互行为 | 具体的用户操作和系统响应 | 写成功能说明（应描述交互） |
| 规则落地 | 引用规则 ID + 用户感知表现 | 重复定义规则逻辑（应只引用+转译） |
| 数据绑定 | 引用实体+字段+接口 | 重新定义字段结构（应引用 class.md） |
| 状态影响 | 引用状态机转换 | 编造状态转换（应来自 states/） |
| 场景回溯 | 引用 L4 scenario 文件名 | 写模糊描述（应精确到文件名） |

## 文件九：05-场景功能/CLAUDE.md — 约束摘要

### 标准格式

```markdown
# L5 约束摘要
{200字以内：功能架构要点、核心规则要点}

## 强制回溯提醒
- L2 终端分工表（每个终端的页面差异必须与之一致）
- L3 痛点列表（每个痛点必须有对应功能解决方案）

## 目录结构说明
- models/：跨终端共享的领域模型（类图、状态机、规则）
- {终端名}/：终端交付物（page-flow + api-view + pages）
- 规则定义在 models/rules/ 中，规则落地在各终端的 spec.md 和 page-flow.md 中
- plan.md：执行计划和进度追踪（执行记录，供回溯参考）
```

### 编写规则

| 字段 | 规则 |
|------|------|
| 约束摘要 | 从 index.md 提取，200 字以内 |
| 强制回溯提醒 | 固定包含：L2 终端分工表 + L3 痛点列表 |
| 目录结构说明 | 列出 models/ + 终端目录 + plan.md 的用途 |
