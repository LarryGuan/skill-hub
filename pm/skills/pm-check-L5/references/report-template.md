## 报告规则

| 规则 | 说明 |
|------|------|
| 结论判定（整体） | 全部 28 项 ✅ → 通过；任何 1 项 ❌ → 不通过 |
| 结论判定（阶段） | 该阶段全部检查项 ✅ → 通过；任何 1 项 ❌ → 不通过 |
| 问题描述 | ❌ 必须附上具体文件位置和不符合的原因 |
| 场景覆盖专项 | 整体检查必须包含，列出 L4 每个核心场景的覆盖状态 |
| 痛点覆盖专项 | 整体检查必须包含，列出 L3 每个核心痛点的解决状态 |
| 运行闭环专项 | 整体检查必须包含，列出每个核心实体的生命周期完整性 |
| 改进建议 | 只针对不通过项，给出具体可操作的修复方向 |
| 不修改 | 检查报告只输出文本，不写入项目文件 |
| 强制修正（整体） | A1（场景覆盖）、C1（四步法）、D2（痛点覆盖）、D4（运行闭环）不通过必须修正 |
| 强制修正（阶段） | P1-1（实体覆盖）、P2-1（四步法）、P2-2（功能复用）、P3-1（API覆盖）不通过必须修正 |
