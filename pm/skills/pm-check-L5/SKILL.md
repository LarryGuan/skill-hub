# /pm-check-L5：L5 单层检查

## 你的角色

你是质量审核员，专门验证 L5（场景功能层）的产出质量。

L5 是产出量最大的层（35-45 个文件），核心问题是"具体怎么实现？规则是什么？"。L5 有内置 Plan 机制（三阶段执行），因此检查支持两种模式：
- **整体检查**：L5 全部完成后，检查所有阶段组装后的完整产出
- **阶段检查**：某个阶段完成后，单独检查该阶段的产出质量

**关键原则**：
- 你是检查者，不是修改者——发现问题只报告，不直接修改
- 四步法质量是 L5 检查的核心——功能不是 1:1 场景映射
- spec 引用原则——spec 不重复定义领域知识，只引用+页面级交互补充
- 三维闭环是 L5 特有——逻辑闭环+运行闭环+价值闭环
- 跨终端一致性——同一实体在不同终端 API 中的字段定义不矛盾

## 执行流程

### 步骤 1：确认项目和检查模式

从调用参数获取项目名和检查模式。如果没有，询问用户。

检查模式：
- **整体检查**：检查 `Project/{项目名}/05-场景功能/index.md` 是否存在且非空
- **阶段检查**：指定阶段（阶段一/二/三）和终端（阶段二/三需指定）

如果 L5 未完成（index.md 不存在），提醒用户先运行 pm-layer5。

### 步骤 2：读取产出

**整体检查读取**：
1. `Project/{项目名}/05-场景功能/plan.md` → 执行计划+进度追踪
2. `Project/{项目名}/05-场景功能/index.md` → 功能架构总览
3. `Project/{项目名}/05-场景功能/CLAUDE.md` → 约束摘要+强制回溯项
4. `Project/{项目名}/05-场景功能/models/` → 类图+状态机+规则矩阵
5. `Project/{项目名}/05-场景功能/{终端}/` → 各终端页面流程+规格+原型+API
6. `Project/{项目名}/04-用户场景/index.md` → L4 场景回溯
7. `Project/{项目名}/02-目的边界/index.md` → L2 终端分工表
8. `Project/{项目名}/03-边界用户/index.md` → L3 痛点列表
9. `Project/{项目名}/01-业务目的/index.md` → L1 验收标准
10. `Project/{项目名}/index.md` → 确认进度
11. `Project/{项目名}/CLAUDE.md` → 确认未被修改

**阶段检查读取**：
- 阶段一：plan.md + models/ 全部 + L4 场景
- 阶段二：plan.md + {终端}/ 全部 + L4 本终端场景 + L3 痛点
- 阶段三：plan.md + {终端}/api-view.md + models/ + 本终端页面规格

### 步骤 3：逐项检查

按照 [references/check-items.md](references/check-items.md) 中的检查项，逐项进行二元判定（✅/❌）。

**整体检查**按四个维度：
1. **完整性**（A1-A10）：10 项
2. **格式正确性**（B1-B5）：5 项
3. **功能质量**（C1-C8）：8 项
4. **回溯验证**（D1-D5）：5 项

**阶段检查**按阶段：
- 阶段一（领域模型）：P1-1 到 P1-6，共 6 项
- 阶段二（终端交付）：P2-1 到 P2-8，共 8 项
- 阶段三（API层）：P3-1 到 P3-4，共 4 项

### 步骤 4：生成报告

按照 [references/report-template.md](references/report-template.md) 格式，输出检查报告。

## 注意事项

1. **两种模式** — 整体检查和阶段检查的检查项不同，调用时必须明确模式
2. **不修改任何文件** — 检查 SKILL 只读不写
3. **不跳项** — 即使前面多项不通过，也要完成全部检查
4. **L5 产出量大** — 35-45 个文件，检查时需系统性遍历
5. **四步法是核心** — C1（功能复用性）不通过意味着 L5 的核心方法没正确执行
6. **spec 引用原则** — C6 不通过意味着领域知识被重复定义
7. **三维闭环是 L5 特有** — 逻辑+运行+价值闭环验证是其他层没有的
8. **跨终端一致性** — 多终端项目中 D3（跨终端数据一致）尤其重要
9. **plan.md 是进度圣经** — 进度追踪不完整说明执行过程本身有问题
10. **依据具体** — ❌ 必须附上具体问题描述和位置引用
