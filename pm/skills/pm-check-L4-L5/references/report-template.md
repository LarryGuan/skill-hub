## 报告规则

| 规则 | 说明 |
|------|------|
| 结论判定 | 全部 ✅ → 通过；有 ⚠️ 无 ❌ → 有风险；有 ❌ → 不通过 |
| C1 优先级 | C1（新功能）❌ 必须回溯 L4，是最严重的断裂 |
| B1 优先级 | B1（四步法）❌ 意味着核心方法论未执行，必须重新执行 L5 |
| 三级判定 | ✅ 通过 / ⚠️ 有风险（需人判断） / ❌ 不通过 |
| 专项表必填 | 场景覆盖、痛点解决、运行闭环三个专项表必须完整填写 |
| 闭环检查 | 运行闭环专项是 L5 特有检查，重点关注核心实体数据流 |
| 风险提示 | ⚠️ 项单独列出，附建议处理方式 |
| 改进建议 | 只针对 ❌ 项，给出可操作的改进方向 |
| 不修改 | 检查报告只输出文本，不写入项目文件 |
| 强制回溯 | 必须回溯 L2 终端分工表 + L3 痛点列表，验证 L5 的强制回溯是否正确执行 |
