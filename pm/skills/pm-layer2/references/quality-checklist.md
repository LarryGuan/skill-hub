# L2 质量检查清单

## 完整性检查

- [ ] 主体链接图是否完整？（所有参与主体都已识别，事件覆盖核心业务流）
- [ ] 主体链接图所有事件是否有终端归属标注？（不能有事件没标终端）
- [ ] 终端分工表是否完整？（每个主体的终端类型和能力边界都已描述）
- [ ] 终端场景清单是否覆盖所有终端？（每个终端都有事件序列）
- [ ] 终端场景清单的事件是否与主体链接图一致？（序号和描述要对得上）
- [ ] 边界表是否完整？（主体链接图中的每条事件都有纳入/不纳入判定）
- [ ] 边界表是否平衡？（有"做什么"也有"不做什么"，不是全部纳入）
- [ ] 类图初版是否包含核心实体和关系？（不需要完整字段，但骨架要对）
- [ ] 用例图初版是否覆盖纳入事件？（不纳入的事件不应出现在用例图中）
- [ ] 产品上下文·扩展是否在 L1 种子基础上增加了"做什么不做什么"？

## 质量检查

- [ ] 主体粒度是否合适？（不能太细如"张三"，也不能太粗如"用户"）
- [ ] 事件描述是否是业务级别？（不应包含"点击按钮"等实现细节）
- [ ] 序号是否按事务发生过程排列？（序号代表业务推进顺序）
- [ ] 终端归属是否合理？（事件发生在什么终端上要符合业务逻辑）
- [ ] 边界判定的原因是否具体？（不能是"需要"/"不需要"这种空泛描述）
- [ ] 类图初版的实体是否从主体链接图推导而来？（不能凭空出现）
- [ ] 用例图初版的用例是否与边界表"纳入"事件一一对应？

## 终端覆盖检查

- [ ] 每个终端至少有 2 个事件？（只有 1 个事件的终端可能是多余的）
- [ ] 每个主体至少在 1 个终端上有操作？（不在任何终端操作的主体可能不需要）
- [ ] 跨终端的事件是否在所有相关终端的场景清单中出现？
- [ ] 终端场景清单的定位描述是否准确？（核心流程/管理端/运营端）

## 常见错误自查

- [ ] 是否避免了遗漏间接主体？（如物流、财务、审核员等边缘角色）
- [ ] 是否避免了"全纳入"？（什么都不排除 = 没有边界）
- [ ] 是否避免了终端类型遗漏？（如忘了 App 或 H5）
- [ ] 是否避免了主体链接图缺少方向？（每条连线必须有方向，代表 A 对 B 做了什么）
- [ ] 是否避免了事件序号跳跃？（序号应该连续，不能有断点）
- [ ] 是否避免了类图初版太详细？（L2 只需骨架，L5 才细化）

## 框架检查

- [ ] 02-目的边界/CLAUDE.md 已生成？（约束摘要 + 强制回溯项 + 下游须知）
- [ ] 02-目的边界/CLAUDE.md 的约束摘要控制在 200 字以内？
- [ ] 项目 index.md 的 L2 进度标记已更新为 [x]？
- [ ] 项目根 CLAUDE.md 未被修改？（框架宪法，只读）
- [ ] 产出中不包含 L1 应有的内容？（手段 vs 目的检查，L2 不重复 L1 的工作）

## 写入前

以上所有检查项必须通过。任何一项不通过都必须补全后才能写入文件。
