# L1↔L2 跨层检查报告模板

```markdown
# L1↔L2 跨层检查报告 — {项目名}

**检查时间**：{时间}
**结论**：✅ 通过 / ⚠️ 有风险 / ❌ 不通过

## 摘要
- 约束传递：{通过数}/4
- 一致性：{通过数}/4
- 断裂检测：{通过数}/3
- **总计：{通过数}/11**
- ⚠️ 有风险项：{列出}

## 详细判定

### A. 约束传递
- [A1] 核心目的→业务事件：✅ / ❌
  - L1 核心目的：{列出}
  - L2 对应事件：{列出}
  - 缺失项：{若有}
- [A2] 验收标准→纳入事件：✅ / ❌ {问题描述}
- [A3] 核心特征→边界体现：✅ / ❌ {问题描述}
- [A4] 下游须知遵守：✅ / ❌ {问题描述}

### B. 一致性
- [B1] 上下文 L1 部分一致：✅ / ❌ {问题描述}
- [B2] 边界不矛盾：✅ / ❌ {问题描述}
- [B3] 事件范围匹配：✅ / ⚠️ / ❌ {问题描述}
- [B4] 第一性原理不偏离：✅ / ❌ {问题描述}

### C. 断裂检测
- [C1] 新概念合理性：✅ / ⚠️ / ❌
  - 新概念列表：{列出}
  - 合理性判断：{逐条}
- [C2] 场景遗漏：✅ / ❌ {遗漏描述}
- [C3] 验收可验证性：✅ / ❌ {哪些标准无法验证}

## 风险提示（⚠️ 项）
1. {具体风险描述 + 建议处理方式}
2. ...

## 改进建议（仅 ❌ 项）
1. {具体改进建议}
2. ...
```

## 报告规则

| 规则 | 说明 |
|------|------|
| 结论判定 | 全部 ✅ → 通过；有 ⚠️ 无 ❌ → 有风险；有 ❌ → 不通过 |
| 三级判定 | ✅ 通过 / ⚠️ 有风险（需人判断） / ❌ 不通过 |
| 风险提示 | ⚠️ 项单独列出，附建议处理方式 |
| 改进建议 | 只针对 ❌ 项，给出可操作的改进方向 |
| 不修改 | 检查报告只输出文本，不写入项目文件 |
