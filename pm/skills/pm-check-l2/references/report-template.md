# L2 检查报告模板

```markdown
# L2 检查报告 — {项目名}

**检查时间**：{时间}
**结论**：✅ 通过 / ❌ 不通过

## 摘要
- 完整性：{通过数}/10
- 格式正确性：{通过数}/4
- 终端覆盖：{通过数}/4
- 内容质量：{通过数}/7
- **总计：{通过数}/25**

## 详细判定

### A. 完整性
- [A1] 主体链接图完整：✅ / ❌ {问题描述}
- [A2] 事件终端标注：✅ / ❌ {问题描述}
- [A3] 终端分工表：✅ / ❌ {问题描述}
- [A4] 终端场景清单覆盖：✅ / ❌ {问题描述}
- [A5] 场景清单一致性：✅ / ❌ {问题描述}
- [A6] 边界表完整：✅ / ❌ {问题描述}
- [A7] 边界表平衡：✅ / ❌ {问题描述}
- [A8] 类图初版：✅ / ❌ {问题描述}
- [A9] 用例图初版：✅ / ❌ {问题描述}
- [A10] 产品上下文·扩展：✅ / ❌ {问题描述}

### B. 格式正确性
- [B1] CLAUDE.md 已生成：✅ / ❌ {问题描述}
- [B2] CLAUDE.md 约束摘要长度：✅ / ❌ {问题描述}
- [B3] 进度标记：✅ / ❌ {问题描述}
- [B4] 框架宪法保护：✅ / ❌ {问题描述}

### C. 终端覆盖
- [C1] 终端事件密度：✅ / ❌ {问题描述}
- [C2] 主体终端覆盖：✅ / ❌ {问题描述}
- [C3] 跨终端事件：✅ / ❌ {问题描述}
- [C4] 场景清单定位：✅ / ❌ {问题描述}

### D. 内容质量
- [D1] 主体粒度：✅ / ❌ {问题描述}
- [D2] 事件描述级别：✅ / ❌ {问题描述}
- [D3] 事件序号：✅ / ❌ {问题描述}
- [D4] 终端归属合理性：✅ / ❌ {问题描述}
- [D5] 边界判定原因：✅ / ❌ {问题描述}
- [D6] 类图推导来源：✅ / ❌ {问题描述}
- [D7] 用例图对应：✅ / ❌ {问题描述}

## 改进建议（仅不通过项）
1. {具体改进建议}
2. ...
```

## 报告规则

| 规则 | 说明 |
|------|------|
| 结论判定 | 全部 25 项 ✅ → 通过；任何 1 项 ❌ → 不通过 |
| 问题描述 | ❌ 必须附上具体位置和不符合的原因 |
| 改进建议 | 只针对不通过项，给出可操作的改进方向 |
| 不修改 | 检查报告只输出文本，不写入项目文件 |
| 终端重点 | 终端覆盖（C1-C4）不通过时优先改进 |
