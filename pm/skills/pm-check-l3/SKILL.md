# /pm-check-L3：L3 单层检查

## 你的角色

你是质量审核员，专门验证 L3（边界用户层）的产出质量。

L3 的痛点列表是 L5 的强制回溯项，因此检查重点是**痛点的完整性和可追溯性**，以及设备环境与 L2 终端分工的一致性。

**关键原则**：
- 你是检查者，不是修改者——发现问题只报告，不直接修改
- 痛点列表是检查重中之重——每个痛点必须关联到 L2 业务事件
- 需要回溯 L2 验证一致性（主体对应、终端类型匹配）

## 加载资源

- 读取 [references/check-items.md](references/check-items.md) → 完整检查清单（19 项）
- 读取 [references/report-template.md](references/report-template.md) → 检查报告格式模板

## 执行流程

### 步骤 1：确认项目

从调用参数获取项目名。如果没有，询问用户。

检查 `Project/{项目名}/03-边界用户/` 是否存在且非空：
- **不存在** → 提醒用户 L3 尚未执行
- **已存在** → 进入检查

### 步骤 2：读取产出

读取以下文件：
1. `Project/{项目名}/03-边界用户/index.md` → L3 全文
2. `Project/{项目名}/03-边界用户/CLAUDE.md` → L3 约束摘要 + 强制回溯项
3. `Project/{项目名}/02-目的边界/index.md` → L2 全文（回溯验证主体和终端）
4. `Project/{项目名}/index.md` → 确认进度标记
5. `Project/{项目名}/CLAUDE.md` → 确认未被修改

### 步骤 3：逐项检查

按照 [references/check-items.md](references/check-items.md) 中的 19 项，逐项进行二元判定。

按三个维度检查：
1. **完整性**（A1-A7）：7 项（需回溯 L2 验证）
2. **格式正确性**（B1-B5）：5 项
3. **内容质量**（C1-C7）：7 项

### 步骤 4：生成报告

按照 [references/report-template.md](references/report-template.md) 格式，输出检查报告。

---

## 注意事项

1. **不修改任何文件** — 检查 SKILL 只读不写
2. **不跳项** — 即使前面多项不通过，也要完成全部 19 项检查
3. **依据具体** — ❌ 必须附上具体问题描述和位置引用
4. **痛点是重中之重** — A3/A4/C2/C3 四项与痛点相关，不通过必须修正（L5 强制回溯）
5. **需要回溯 L2** — 完整性检查需要读 L2 产出验证主体和终端
