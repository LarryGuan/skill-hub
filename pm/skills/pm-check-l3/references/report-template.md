# L3 检查报告模板

```markdown
# L3 检查报告 — {项目名}

**检查时间**：{时间}
**结论**：✅ 通过 / ❌ 不通过

## 摘要
- 完整性：{通过数}/7
- 格式正确性：{通过数}/5
- 内容质量：{通过数}/7
- **总计：{通过数}/19**

## 详细判定

### A. 完整性
- [A1] 主体全覆盖：✅ / ❌ {问题描述}
- [A2] 画像要素完整：✅ / ❌ {问题描述}
- [A3] 痛点关联 L2 事件：✅ / ❌ {问题描述}
- [A4] 痛点主体覆盖：✅ / ❌ {问题描述}
- [A5] 环境主体覆盖：✅ / ❌ {问题描述}
- [A6] 环境终端一致：✅ / ❌ {问题描述}
- [A7] 产品上下文三维：✅ / ❌ {问题描述}

### B. 格式正确性
- [B1] CLAUDE.md 已生成：✅ / ❌ {问题描述}
- [B2] CLAUDE.md 约束摘要长度：✅ / ❌ {问题描述}
- [B3] 强制回溯项完整：✅ / ❌ {问题描述}
- [B4] 进度标记：✅ / ❌ {问题描述}
- [B5] 框架宪法保护：✅ / ❌ {问题描述}

### C. 内容质量
- [C1] 画像特征相关性：✅ / ❌ {问题描述}
- [C2] 痛点具体可感知：✅ / ❌ {问题描述}
- [C3] 痛点严重程度分层：✅ / ❌ {问题描述}
- [C4] 环境约束具体：✅ / ❌ {问题描述}
- [C5] 上下文 L1/L2 一致：✅ / ❌ {问题描述}
- [C6] 上下文 L3 纯净：✅ / ❌ {问题描述}
- [C7] 间接主体无遗漏：✅ / ❌ {问题描述}

## 痛点专项评估
- L2 事件总数：{n}
- 痛点覆盖事件数：{n}
- 覆盖率：{n}%
- 未覆盖事件：{列出}

## 改进建议（仅不通过项）
1. {具体改进建议}
2. ...
```

## 报告规则

| 规则 | 说明 |
|------|------|
| 结论判定 | 全部 19 项 ✅ → 通过；任何 1 项 ❌ → 不通过 |
| 问题描述 | ❌ 必须附上具体位置和不符合的原因 |
| 痛点专项 | 单独列出痛点覆盖评估，便于 L5 回溯 |
| 改进建议 | 只针对不通过项，痛点相关不通过优先改进 |
| 不修改 | 检查报告只输出文本，不写入项目文件 |
| 强制修正 | A3/A4（痛点关联和覆盖）不通过必须修正 |
