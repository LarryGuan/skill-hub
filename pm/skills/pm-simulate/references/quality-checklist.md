# 模拟走查质量检查清单

## 1. 走查说明完整性（1走查说明.md）

- [ ] 是否列出了所有读取的 L3/L4/L5 文件路径？
- [ ] 是否记录了原始素材摘要（过滤前）？
- [ ] 是否有术语过滤对照表（原始术语→日常语言）？
- [ ] 角色卡是否不包含产品术语？（不能出现"模块"、"API"、"实体"、"功能"等词）
- [ ] 角色卡是否不包含设计意图描述？
- [ ] 任务目标是否用日常语言？（不能出现"使用XX功能"）
- [ ] Sub-agent prompt 全文是否完整记录？
- [ ] Sub-agent prompt 是否不包含 L1-L4 的设计推导内容？
- [ ] Sub-agent prompt 是否不包含 page-flow？（页面跳转设计）
- [ ] Sub-agent prompt 是否包含本终端所有页面？（V2 核心要求）

## 2. 走查记录保真性（2走查记录.md）

- [ ] 走查记录内容是否为 sub-agent 原始输出？（主 Agent 未修改内容）
- [ ] 是否有来源标记？（sub-agent直接写入 | 主Agent从tool result回填）
- [ ] 如果是回填，来源标记是否为"主Agent从tool result回填"？
- [ ] Sub-agent 是否用日常语言描述看到的内容？
- [ ] Sub-agent 是否报告了真实感受（不是每步都"满意"）？

## 3. 写入可靠性检查

- [ ] 主 Agent 是否在调用 sub-agent 前创建了目标目录？
- [ ] 主 Agent 是否在调用 sub-agent 前写了 stub 文件？
- [ ] 调用 sub-agent 后，主 Agent 是否检查了 2走查记录.md 是否被写入？
- [ ] 如果是回填，主 Agent 是否在 3走查报告中说明了回填原因？

## 4. 覆盖性检查

- [ ] 每个人类用户角色是否至少有一个模拟走查？
- [ ] 每个终端是否至少有一个模拟走查？
- [ ] 每个关键场景是否被至少一个角色走查过？
- [ ] 不同角色的视角是否真的不同？（不能所有角色看到的问题一样）

## 5. 走查质量检查

- [ ] Sub-agent 是否从"打开系统/进入产品"开始？（不能跳过入口直接进入功能）
- [ ] 走查是否到达了目标或发现了死路？（不能半途而废）
- [ ] Sub-agent 是否自由选择探索路径？（没有被引导走特定路线）
- [ ] Sub-agent 是否探索了多个页面？（不是只看了一个页面就结束）

## 6. 走查报告质量（3走查报告.md）

### 报告结构
- [ ] 是否引用了走查说明和走查记录文件？
- [ ] 走查过程是否从磁盘读取的走查记录整理而来？（非凭空编写）
- [ ] 是否有走查顺利度评分（1-5 分）？

### 质量评估
- [ ] 是否评价了 sub-agent 的角色代入度？
- [ ] 是否评价了 sub-agent 的探索充分度？
- [ ] 是否评价了 sub-agent 的感受真实度？
- [ ] 是否指出了跳戏时刻（如果有的话）？

### 问题报告
- [ ] 每个问题是否从四种类型（迷惑🟡/死路🔴/摩擦🟠/错失🔵）中选择了正确的类型？
- [ ] 每个问题是否有严重程度评级？（高/中/低）
- [ ] 每个问题是否标注了涉及的 spec 文件名？
- [ ] 每个问题是否有具体的修改建议？（不能只说"这里不好"）
- [ ] 高严重程度的问题是否真的是死路？（不能把摩擦标成高）
- [ ] 是否有"错失"类型的问题？（这是 V2 自由探索最能发现的）
- [ ] 是否记录了做得好的地方？（不能只有问题）

## 7. 格式检查

- [ ] 文件命名是否符合三文件格式？（1走查说明 / 2走查记录 / 3走查报告）
- [ ] 多终端项目是否按终端拆目录？
- [ ] 三文件是否齐全？（缺少任何一个都不完整）
- [ ] 走查说明中的角色卡变量是否已替换为实际内容？

## 写入前

以上所有检查项必须通过。任何一项不通过都必须补全后才能写入文件。
