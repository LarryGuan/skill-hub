# 模拟走查产出格式模板

## 文件位置

每次走查产出一个三文件组：

```
06-验收测试/
└── simulations/
    └── {终端名}/
        ├── sim-{角色}-{场景}-1走查说明.md   ← 主 Agent 构造过程留底
        ├── sim-{角色}-{场景}-2走查记录.md   ← Sub-agent 原始探索记录
        └── sim-{角色}-{场景}-3走查报告.md   ← 主 Agent 编译的结构化报告
```

多终端项目按终端拆目录（按 L2 定义的终端名建目录，如 mp/、pc/），单终端项目直接放 simulations/ 下。

```markdown

- 走查组合：{角色名} × {终端} × {场景名}
- 对应 L3 画像：{画像主体名}
- 对应 L4 场景：{scenario 文件名}
- 构造时间：{日期}

## 1. 读取记录

本次走查读取的原始文件：

### L3 画像文件
- `03-边界用户/index.md` → 提取了 {哪些画像信息}
- （其他读取的文件列表）

### L4 场景文件
- `04-用户场景/index.md` → 提取了 {哪些场景信息}
- `04-用户场景/{终端}/scenario-{N}.md` → 提取了 {触发条件、目标}
- （其他读取的文件列表）

### L5 页面规格文件
- `05-场景功能/{终端}/pages/{页面A}/spec.md` → 提取了 {用户可感知部分}
- `05-场景功能/{终端}/pages/{页面B}/spec.md` → 提取了 {用户可感知部分}
- （所有本终端页面的列表）

## 2. 原始素材摘要

### 从 L3 提取的画像信息
{原始提取的画像内容，未过滤版本}

### 从 L4 提取的场景信息
{原始提取的触发条件、目标、环境}

### 从 L5 提取的页面信息
{原始提取的页面内容，未过滤版本}

## 3. 术语过滤对照

| 原始术语 | 过滤后（用户可感知描述） |
|---------|----------------------|
| {产品术语，如"产品列表组件"} | {日常语言，如"一排排的水果卡片"} |
| {产品术语} | {日常语言} |
| ... | ... |

## 4. 角色卡（最终版）

- 我是谁：{日常语言描述，如"我是一个水果批发商，姓王，在广州江南市场有摊位"}

## 5. 任务目标（最终版）

{从 L4 触发条件转化的日常语言目标}

## 6. Sub-agent Prompt 全文

{完整的、发送给 sub-agent 的 prompt，包含：}
- 角色代入指令
- 角色卡
- 设备环境
- 目标
- 本终端所有页面的用户可感知描述
- 行为指引 + 输出格式要求
```

## 文件二：2走查记录.md（Sub-agent）

```markdown
# 走查记录：{角色名} x {终端} x {场景名}

> **来源标记：** {sub-agent直接写入 | 主Agent从tool result回填}
> 本文件保持 sub-agent 原始输出，主 Agent 不修改内容。

---

{sub-agent 的原始探索记录，格式自由，无强制格式要求}
```

### 回填说明

当 sub-agent 未能成功写入本文件时（如文件仅含 stub 头部或不存在），主 Agent 从 tool result 中提取 sub-agent 的原始输出，写入本文件。此时来源标记为"主Agent从tool result回填"，但文件内容仍为 sub-agent 的原始输出，主 Agent 不做任何内容修改。

**关键要求**：
- Sub-agent 自由输出，无强制格式要求
- 主 Agent **不修改**本文件内容，在走查报告中引用和整理

## 文件三：3走查报告.md（主 Agent）

```markdown
# 走查报告：{角色名} x {终端} x {场景名}

## 走查概况
- 走查组合：{角色名} × {终端} × {场景名}
- 走查说明：sim-{角色}-{场景}-1走查说明.md
- 走查记录：sim-{角色}-{场景}-2走查记录.md

## 走查顺利度
- 评分：{1-5 分，5 分最顺畅}
- 完成目标：{是/否}
- 总耗时感：{很快/还行/太慢了}

## 走查过程整理

### 步骤 1：{标题}
- 用户看到：{整理后的描述}
- 用户想做：{整理后的描述}
- 用户操作了：{整理后的描述}
- 系统反馈：{整理后的描述}
- 用户感受：{满意😊 / 困惑😕 / 沮丧😞 / 着急😤}
- ⚠️ 问题：{如果有，标注类型和描述}

### 步骤 2：...
（逐步骤整理完整探索过程）

## 走查质量评估

### 角色代入度
- Sub-agent 是否成功代入角色？{评价}
- 是否有跳戏时刻（用了设计知识）？{具体说明}

### 探索充分度
- 是否探索了多个页面？{说明}
- 是否从"打开系统"开始？{说明}
- 是否到达目标或发现死路？{说明}
- 是否自由选择路径（非被引导）？{说明}

### 感受真实度
- 是否报告了真实感受（非全部"满意"）？{评价}
- 问题报告是否具体？{评价}

## 发现的问题

| # | 类型 | 描述 | 严重程度 | 涉及 spec | 建议修改 |
|---|------|------|----------|-----------|----------|
| 1 | 迷惑🟡/死路🔴/摩擦🟠/错失🔵 | {具体描述} | 高/中/低 | {spec 文件名} | {怎么改} |
| 2 | 迷惑🟡/死路🔴/摩擦🟠/错失🔵 | {具体描述} | 高/中/低 | {spec 文件名} | {怎么改} |

## 做得好的地方

{记录顺畅的、直觉的部分，帮助确认设计决策的正确性}

## 总结评定

{本次走查的整体评价、关键发现、建议关注的重点}
```

---

## 问题类型说明

| 类型 | 图标 | 定义 | 严重程度参考 |
|------|------|------|-------------|
| 迷惑 | 🟡 | 用户不理解为什么要这样操作，或者找不到操作入口 | 中 |
| 死路 | 🔴 | 用户想做的事但系统不支持，无法继续 | 高 |
| 摩擦 | 🟠 | 能做但步骤太多、不直觉、需要反复思考 | 低~中 |
| 错失 | 🔵 | 用户期待有但系统没有的功能 | 视情况 |

---

## 字段说明

| 字段 | 所在文件 | 要求 | 常见错误 |
|------|---------|------|----------|
| 读取记录 | 1走查说明 | 必须列出所有读取的 L3/L4/L5 文件 | 漏掉某些页面 spec |
| 术语过滤对照 | 1走查说明 | 每个过滤都要记录原文和转化后 | 没有记录过滤过程 |
| 角色卡 | 1走查说明 | 必须用日常语言，不含产品术语 | "作为一个终端用户"太抽象 |
| sub-agent prompt 全文 | 1走查说明 | 完整记录发送给 sub-agent 的内容 | 只写了摘要 |
| 走查记录内容 | 2走查记录 | 自由格式，无强制格式要求 | 主 Agent 格式化了原始记录 |
| 来源标记 | 2走查记录 | 必须标注，值为"sub-agent直接写入"或"主Agent从tool result回填"之一 | 未标注来源标记 |
| 走查质量评估 | 3走查报告 | 必须评价 sub-agent 的走查质量 | 只整理问题不评估质量 |
| 问题类型 | 3走查报告 | 必须从四种类型中选 | 自己发明新类型 |
| 严重程度 | 3走查报告 | 死路=高，迷惑=中，摩擦=低~中 | 所有都标高 |
| 涉及 spec | 3走查报告 | 必须标注对应的 spec 文件名 | 写了"全局"但没有具体文件 |
| 建议修改 | 3走查报告 | 必须具体可操作 | "改进体验"太笼统 |
| 做得好的地方 | 3走查报告 | 必须有，不能全是问题 | 完全忽略好的体验 |
