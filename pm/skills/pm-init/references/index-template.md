# 项目 index.md 模板

pm-init 生成的项目 index.md 必须遵循以下模板：

```markdown
# {项目名}
{一句话描述}

## 进度
- [x] Init
- [ ] L1 业务目的
- [ ] L2 目的边界
- [ ] L3 边界用户
- [ ] L4 用户场景
- [ ] L5 场景功能

## 原始想法
{从用户采集的原始想法，完整保留}

## 目录结构
- [01-业务目的](01-业务目的/) — 为什么做这件事
- [02-目的边界](02-目的边界/) — 做什么、不做什么
- [03-边界用户](03-边界用户/) — 谁参与这件事
- [04-用户场景](04-用户场景/) — 参与者做什么
- [05-场景功能](05-场景功能/) — 具体功能设计
- [capture](capture/) — 产品认知记录（决策、洞察、框架认知）
```

## 编写规则

| 字段 | 规则 |
|------|------|
| 标题 | 项目名称，与 CLAUDE.md 一致 |
| 一句话描述 | 与 CLAUDE.md 一致，不超过 50 字 |
| 进度 | 固定 6 项，每个 SKILL 完成后更新对应标记为 `[x]` |
| 原始想法 | 从步骤 1 采集的用户原始想法，完整保留不做加工 |
| 目录结构 | 固定 6 项，使用 markdown 链接 |

## 进度更新规则

- 每个 SKILL 完成后，更新对应层的进度标记：`- [ ]` → `- [x]`
- 进度标记是 SKILL 可写的少数内容之一（另一处是各层 index.md 产出）
- 如果 SKILL 执行后回头发现问题需要回退，进度标记也应回退
