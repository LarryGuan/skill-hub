# 项目骨架目录模板

## 初始化创建的目录结构

pm-init 创建以下骨架：

```
{项目名}/
├── CLAUDE.md                    # 项目级规则（全局约束）
├── index.md                     # 项目总览 + 进度
├── capture/
│   └── index.md                 # 认知记录（仅标题，pm-capture 补全）
├── 01-业务目的/                 # 空目录
├── 02-目的边界/                 # 空目录
├── 03-边界用户/                 # 空目录
├── 04-用户场景/                 # 空目录
└── 05-场景功能/                 # 空目录
```

**注意**：04 和 05 的终端子目录在 L4/L5 SKILL 执行时创建，不在初始化时创建。

## 完成态目录结构（参考）

```
{项目名}/
├── CLAUDE.md                    # 项目级规则（全局约束）
├── index.md                     # 项目总览 + 进度
├── capture/                    # 产品认知记录（决策、洞察、框架认知）
│   ├── CLAUDE.md               # 目录使用规范
│   ├── index.md                # 总索引（高影响区 + 按类型索引）
│   ├── decisions.md            # 决策记录
│   ├── insights.md             # 洞察记录
│   └── frameworks.md           # 框架认知记录
├── 01-业务目的/
│   ├── CLAUDE.md                # L1 规则（约束摘要 + 下游须知）
│   └── index.md                 # 第一性原理、核心目的、验收标准
├── 02-目的边界/
│   ├── CLAUDE.md                # L2 规则（约束摘要 + 终端分工 + 下游须知）
│   └── index.md                 # 边界表、终端分工表、业务模型图、类图初版、用例图初版
├── 03-边界用户/
│   ├── CLAUDE.md                # L3 规则（约束摘要 + 痛点列表 + 下游须知）
│   └── index.md                 # 用户画像总览、主体说明、使用环境、产品上下文完成版
├── 04-用户场景/
│   ├── CLAUDE.md                # L4 规则（约束摘要 + 场景交叉关系 + 下游须知）
│   ├── index.md                 # 场景总览 + 交叉分析
│   └── {终端名}/                # 按终端组织（即使只有1个终端也保持此结构）
│       ├── scenario-{名称1}.md  # 场景故事
│       └── scenario-{名称2}.md
└── 05-场景功能/
    ├── CLAUDE.md                # L5 规则（约束摘要 + 功能架构要点 + 强制回溯提醒）
    ├── index.md                 # 功能架构总览（终端→模块→功能→功能点的完整挂载）
    ├── models/                  # 领域模型（跨终端共享）
    │   ├── class.md             # 完整类图
    │   ├── states/              # 状态机图（按实体，一个文件一个实体）
    │   │   ├── {实体名1}.md
    │   │   └── {实体名2}.md
    │   └── rules/               # 业务规则矩阵（按领域）
    │       ├── {领域1}-rules.md
    │       └── {领域2}-rules.md
    └── {终端名}/                # 终端交付物（即使只有1个终端也保持此结构）
        ├── page-flow.md         # 页面流程图（Mermaid）
        ├── api-view.md          # API 接口视图（本终端的数据封装）
        └── pages/
            └── {页面名}/
                ├── spec.md      # 页面功能规格（交互 + 规则落地 + 数据绑定）
                └── wireframe.md # 页面原型（ASCII 或低保真线框）
```

## 目录结构要点

- 04 和 05 始终按终端分层，即使只有 1 个终端也保持此结构（与 L2 终端分工表对齐）
- 05 的 models/ 在根级，不在终端下——类图、状态机、规则矩阵是领域级的，跨终端共享
- 05 的终端目录下每个页面有 spec.md + wireframe.md，spec 引用 models/ 中的产出
- 规则定义在 models/rules/ 中，规则落地（用户可感知的交互行为）写在各终端的 spec.md 和 page-flow.md 中
