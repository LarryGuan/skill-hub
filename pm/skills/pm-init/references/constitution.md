# 项目宪法规则

每个项目的 CLAUDE.md 必须包含以下宪法规则，确保 SKILL 体系在项目中正确运行。

## 项目目录规则

1. 项目目录结构固定为 5 层（01~05）+ capture/，不可自行增减
2. 04 和 05 始终按终端分层，即使只有 1 个终端也保持此结构
3. 各层 SKILL 只写入自己负责的目录，不跨界修改

## 进度管理

- 进度在项目根 CLAUDE.md 中跟踪
- 每个 SKILL 完成后更新对应层的进度标记（`- [x]`）
- 进度项固定为 6 项：Init / L1 / L2 / L3 / L4 / L5

## CLAUDE.md 规则

- 每层目录下的 CLAUDE.md 由该层 SKILL 生成/更新
- 项目根 CLAUDE.md 由 pm-init 生成，由后续 SKILL 更新进度
- 约束摘要控制在 200 字以内
- 强制回溯项必须明确列出

## 层间信息传递

- 紧邻上层读全文，隔一层读摘要
- L2 终端分工表：L3/L4/L5 必须全文回溯
- L3 痛点列表：L5 必须全文回溯
