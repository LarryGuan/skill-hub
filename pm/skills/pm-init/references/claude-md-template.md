# 项目根 CLAUDE.md 模板

pm-init 生成的项目根 CLAUDE.md 必须遵循以下模板：

```markdown
# {项目名}
{一句话描述}

## 目录结构
- [01-业务目的](01-业务目的/) — 为什么做这件事
- [02-目的边界](02-目的边界/) — 做什么、不做什么
- [03-边界用户](03-边界用户/) — 谁参与这件事
- [04-用户场景](04-用户场景/) — 参与者做什么
- [05-场景功能](05-场景功能/) — 具体功能设计
- [capture](capture/) — 产品认知记录（决策、洞察、框架认知）

## 框架规则
- 各层 SKILL 只写入自己负责的目录，不跨界修改
- 04 和 05 始终按终端分目录，即使只有 1 个终端也保持此结构
- 紧邻上层读全文，隔一层读摘要
- L2 终端分工表、L3 痛点列表：全文回溯项
- capture/ 由 /pm-capture 独占写入，任何 SKILL 可读取

## 约束来源（只引用，不内嵌）
- 核心目的与验收标准 → [01-业务目的/index.md](01-业务目的/index.md)
- 边界与终端分工 → [02-目的边界/index.md](02-目的边界/index.md)
- 用户画像与痛点 → [03-边界用户/index.md](03-边界用户/index.md)
- 场景故事 → [04-用户场景/index.md](04-用户场景/index.md)
- 功能设计 → [05-场景功能/index.md](05-场景功能/index.md)
- 产品认知记录 → [capture/index.md](capture/index.md)
- 进度跟踪 → [index.md](index.md)
```

## 编写规则

| 字段 | 规则 |
|------|------|
| 标题 | 项目名称，简洁明确 |
| 一句话描述 | 从用户原始想法中提炼，不超过 50 字 |
| 目录结构 | 固定 7 项（5 层 + capture），使用 markdown 链接 |
| 框架规则 | 固定内容，初始化时直接写入，后续不变 |
| 约束来源 | 引用链接指向各层 index.md，不内嵌可变内容 |

## 设计原则

CLAUDE.md 是框架宪法：
- **只放不变的规则和引用链接**，不放可变内容
- 核心约束、进度标记等可变内容由各层 index.md 承载
- 初始化后基本不需要修改（除非新增引用路径）
- 控制总字数在 400 字以内
