# L3↔L4 跨层检查清单（11 项）

## 判定说明

跨层检查使用三级判定：
- **✅ 通过**：符合标准
- **⚠️ 有风险**：存在潜在问题，需要人判断是否可接受
- **❌ 不通过**：明确存在断裂或矛盾

## A. 约束传递（4 项）

### A1. 主体→场景参与者
- **判定标准**：L3 画像总览的所有主体在 L4 场景中作为参与者出现
- **检查方法**：
  1. 从 L3 画像总览提取所有主体名
  2. 从 L4 各终端场景提取所有参与者
  3. 逐个比对
- **✅ 通过**：完全对应
- **❌ 不通过**：L3 有主体在 L4 中缺失

### A2. 痛点→场景痛点命中
- **判定标准**：L3 痛点列表中每个核心痛点在至少一个场景中被"痛点命中"引用
- **检查方法**：
  1. 从 L3 痛点列表提取所有核心痛点
  2. 从 L4 各场景的痛点命中部分提取引用的痛点
  3. 逐个检查覆盖情况
- **✅ 通过**：所有核心痛点被命中
- **❌ 不通过**：有核心痛点未被任何场景命中

### A3. 环境→场景设备环境
- **判定标准**：L3 使用环境的设备类型与 L4 场景中的设备环境描述一致
- **检查方法**：
  1. 从 L3 使用环境提取每个主体的设备类型
  2. 从 L4 各终端场景提取设备环境描述
  3. 逐个比对
- **✅ 通过**：终端类型匹配
- **❌ 不通过**：L4 描述的终端与 L3 不一致

### A4. 强制回溯引用
- **判定标准**：L3 的强制回溯项（痛点列表、使用环境）被 L4 正确引用
- **检查方法**：检查 L4 产出中是否引用了 L3 痛点列表和使用环境的信息
- **✅ 通过**：正确引用
- **❌ 不通过**：未引用或错误引用

## B. 一致性（4 项）

### B1. 行为模式匹配
- **判定标准**：L4 场景中参与者的行为与 L3 画像的行为模式一致
- **检查方法**：
  1. 从 L3 画像提取行为模式描述
  2. 从 L4 场景提取参与者的操作行为
  3. 检查行为模式是否与场景中的参与事件匹配
- **✅ 通过**：行为模式与参与事件匹配
- **⚠️ 有风险**：行为模式未完全覆盖，但差距不大
- **❌ 不通过**：行为模式与参与事件明显矛盾

### B2. 心理特征一致
- **判定标准**：场景中参与者的决策方式与 L3 画像的心理特征匹配
- **检查方法**：
  1. 从 L3 画像提取心理特征描述
  2. 从 L4 场景提取参与者的决策与操作方式
  3. 检查心理特征与场景决策是否一致
- **✅ 通过**：心理特征与场景决策一致
- **⚠️ 有风险**：部分场景决策与心理特征不完全匹配
- **❌ 不通过**：心理特征与场景行为矛盾（如"谨慎型"用户在场景中轻率操作）

### B3. 场景分类合理
- **判定标准**：核心/边缘/异常分类与 L3 痛点严重程度对应
- **检查方法**：
  1. 从 L3 痛点列表提取各痛点的严重程度
  2. 从 L4 场景提取分类标签（核心/边缘/异常）
  3. 检查高频严重痛点对应的核心场景是否被标记为核心
- **✅ 通过**：场景分类合理
- **⚠️ 有风险**：个别场景分类可商榷
- **❌ 不通过**：大量场景分类不合理

### B4. 活动图反映操作路径
- **判定标准**：活动图的操作步骤与 L3 画像描述的行为模式一致
- **检查方法**：
  1. 从 L3 画像提取行为模式中的典型操作路径
  2. 从 L4 场景活动图提取操作步骤
  3. 检查活动图步骤与行为模式是否匹配
- **✅ 通过**：操作路径与行为模式一致
- **⚠️ 有风险**：个别操作步骤与行为模式有差异
- **❌ 不通过**：活动图与行为模式矛盾

## C. 断裂检测（3 项）

### C1. 新事件检测
- **判定标准**：L4 场景是否出现了 L2 主体链接图未识别的新关系事件
- **检查方法**：
  1. 从 L2 主体链接图提取所有关系事件
  2. 从 L4 各场景提取涉及的事件/交互
  3. 检查 L4 是否出现了 L2 未识别的新事件
- **✅ 通过**：无新事件
- **❌ 不通过**：有新事件 → 必须回溯 L2 补充（最严重的断裂）

### C2. 痛点覆盖度
- **判定标准**：L3 核心痛点是否在场景中都有对应环节
- **检查方法**：
  1. 从 L3 痛点列表提取所有核心痛点
  2. 检查每个核心痛点在 L4 场景中是否有对应的痛点命中
- **✅ 通过**：核心痛点都有场景对应
- **⚠️ 有风险**：部分核心痛点无痛点命中
- **❌ 不通过**：大量核心痛点无痛点命中

### C3. 场景与画像矛盾
- **判定标准**：场景中的用户行为是否与 L3 画像矛盾
- **检查方法**：
  1. 从 L3 画像提取用户的关键特征（技能水平、操作习惯、决策方式）
  2. 从 L4 场景提取用户的具体操作与决策
  3. 检查场景行为是否与画像特征矛盾
- **✅ 通过**：无矛盾
- **⚠️ 有风险**：个别行为与画像不完全一致
- **❌ 不通过**：明显矛盾（如新手画像要求复杂操作）
