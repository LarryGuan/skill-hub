# L3↔L4 跨层检查报告模板

```markdown
# L3↔L4 跨层检查报告 — {项目名}

**检查时间**：{时间}
**结论**：✅ 通过 / ⚠️ 有风险 / ❌ 不通过

## 摘要
- 约束传递：{通过数}/4
- 一致性：{通过数}/4
- 断裂检测：{通过数}/3
- **总计：{通过数}/11**
- ⚠️ 有风险项：{列出}
- 🔴 必须修正：{列出 C1 结果}

## 详细判定

### A. 约束传递
- [A1] 主体→场景参与者：✅ / ❌
  - L3 画像主体：{列出}
  - L4 场景参与者：{列出}
  - L4 中缺失的主体：{若有}
- [A2] 痛点→场景痛点命中：✅ / ❌
  - L3 核心痛点：{列出}
  - 未被命中的痛点：{若有}
- [A3] 环境→场景设备环境：✅ / ❌
  - L3 设备类型：{列出}
  - L4 场景设备：{列出}
  - 不一致项：{若有}
- [A4] 强制回溯引用：✅ / ❌ {问题描述}

### B. 一致性
- [B1] 行为模式匹配：✅ / ⚠️ / ❌ {问题描述}
- [B2] 心理特征一致：✅ / ⚠️ / ❌ {问题描述}
- [B3] 场景分类合理：✅ / ⚠️ / ❌ {问题描述}
- [B4] 活动图反映操作路径：✅ / ⚠️ / ❌ {问题描述}

### C. 断裂检测
- [C1] 新事件检测：✅ / ❌
  - 新事件列表：{列出}
  - 处理建议：{必须回溯 L2 补充关系事件}
- [C2] 痛点覆盖度：✅ / ⚠️ / ❌
  - 未覆盖的核心痛点：{列出}
- [C3] 场景与画像矛盾：✅ / ⚠️ / ❌
  - 矛盾点：{列出}

## 严重问题（必须修正）
1. {C1 新事件 或 A3 环境不一致 等严重问题}

## 风险提示（⚠️ 项）
1. {具体风险描述 + 建议处理方式}

## 改进建议（仅 ❌ 项）
1. {具体改进建议}
2. ...
```

## 报告规则

| 规则 | 说明 |
|------|------|
| 结论判定 | 全部 ✅ → 通过；有 ⚠️ 无 ❌ → 有风险；有 ❌ → 不通过 |
| C1 优先级 | C1（新事件）❌ 必须回溯 L2 补充关系事件，是最严重的断裂 |
| 三级判定 | ✅ 通过 / ⚠️ 有风险（需人判断） / ❌ 不通过 |
| 痛点命中 | A2/C2 是 L3→L4 衔接的核心环节，L3 痛点是 L5 强制回溯项 |
| 画像一致性 | B1/B2 是 L3→L4 特有的检查维度，其他跨层不验证 |
| 风险提示 | ⚠️ 项单独列出，附建议处理方式 |
| 改进建议 | 只针对 ❌ 项，给出可操作的改进方向 |
| 不修改 | 检查报告只输出文本，不写入项目文件 |
