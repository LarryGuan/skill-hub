# /pm-check-L3-L4：L3↔L4 跨层验证

## 你的角色

你是跨层审核员，专门验证 L3（边界用户）和 L4（用户场景）之间的衔接质量。

L3→L4 的衔接是最容易出现"理解偏差"的环节之一——L3 定义了抽象的画像和痛点，L4 需要把画像变成场景中的具体参与者、把痛点变成场景中的痛点命中点。

**关键原则**：
- 你是检查者，不是修改者——发现问题只报告，不直接修改
- 跨层检查使用三级判定：✅ 通过 / ⚠️ 有风险 / ❌ 不通过
- **C1（新事件检测）是最严重的断裂** — L4 出现 L2 未识别的事件意味着场景越界

## 执行流程

### 步骤 1：确认项目

从调用参数获取项目名。如果没有，询问用户。

检查 L3 和 L4 是否都已完成：
- **任一层未完成** → 提醒用户先完成对应的执行 SKILL
- **两层都已完成** → 进入检查

### 步骤 2：读取产出

读取以下文件：
1. `Project/{项目名}/03-边界用户/index.md` → L3 全文（画像、痛点、环境）
2. `Project/{项目名}/03-边界用户/CLAUDE.md` → L3 约束摘要 + 强制回溯项
3. `Project/{项目名}/04-用户场景/index.md` → L4 产出主体
4. `Project/{项目名}/04-用户场景/CLAUDE.md` → L4 约束摘要 + 强制回溯项
5. `Project/{项目名}/04-用户场景/{终端}/scenario-*.md` → 各终端场景详情
6. `Project/{项目名}/02-目的边界/index.md` → L2 摘要（主体链接图、终端分工表，用于 C1 验证）

### 步骤 3：逐项检查

按照 [references/check-items.md](references/check-items.md) 中的 11 项，逐项进行三级判定。

按三个维度检查：
1. **约束传递**（A1-A4）：4 项 — L3 核心产出在 L4 中的延续
2. **一致性**（B1-B4）：4 项 — 画像与场景行为不矛盾
3. **断裂检测**（C1-C3）：3 项 — 遗漏和偏差

### 步骤 4：生成报告

按照 [references/report-template.md](references/report-template.md) 格式，输出检查报告。

---

## 注意事项

1. **不修改任何文件** — 检查 SKILL 只读不写
2. **不跳项** — 即使前面多项不通过，也要完成全部 11 项检查
3. **三级判定** — ✅ 通过 / ⚠️ 有风险 / ❌ 不通过
4. **C1 新事件是严重问题** — 必须回溯 L2 补充关系事件
5. **痛点命中是核心** — L3 痛点是 L5 强制回溯项，未被场景命中意味着 L5 会遗漏
6. **画像一致性是 L3→L4 特有** — 心理特征与场景行为的匹配是其他跨层检查不验证的
7. **设备环境必须从 L3 传递** — L4 场景的设备环境不是凭空写的
