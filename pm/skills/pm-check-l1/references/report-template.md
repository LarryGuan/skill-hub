# L1 检查报告模板

```markdown
# L1 检查报告 — {项目名}

**检查时间**：{时间}
**结论**：✅ 通过 / ❌ 不通过

## 摘要
- 完整性：{通过数}/6
- 格式正确性：{通过数}/3
- 内容质量：{通过数}/5
- **总计：{通过数}/14**

## 详细判定

### A. 完整性
- [A1] 背景与痛点：✅ / ❌ {问题描述}
- [A2] 第一性原理：✅ / ❌ {问题描述}
- [A3] 核心特征：✅ / ❌ {问题描述}
- [A4] 核心目的：✅ / ❌ {问题描述}
- [A5] 验收标准：✅ / ❌ {问题描述}
- [A6] 约束摘要：✅ / ❌ {问题描述}

### B. 格式正确性
- [B1] CLAUDE.md 已生成：✅ / ❌ {问题描述}
- [B2] CLAUDE.md 约束摘要长度：✅ / ❌ {问题描述}
- [B3] 进度标记：✅ / ❌ {问题描述}

### C. 内容质量
- [C1] 手段与目的区分：✅ / ❌ {问题描述}
- [C2] 目的粒度适中：✅ / ❌ {问题描述}
- [C3] 验收标准可验证：✅ / ❌ {问题描述}
- [C4] 核心特征覆盖底线：✅ / ❌ {问题描述}
- [C5] 第一性原理纯净：✅ / ❌ {问题描述}

## 改进建议（仅不通过项）
1. {具体改进建议}
2. ...
```

## 报告规则

| 规则 | 说明 |
|------|------|
| 结论判定 | 全部 14 项 ✅ → 通过；任何 1 项 ❌ → 不通过 |
| 问题描述 | ❌ 必须附上具体位置和不符合的原因 |
| 改进建议 | 只针对不通过项，给出可操作的改进方向 |
| 不修改 | 检查报告只输出文本，不写入项目文件 |
