# L1 检查清单（14 项）

## A. 完整性（6 项）

### A1. 背景与痛点
- **判定标准**：有具体的当前问题描述，非泛泛"效率低""体验差"
- **通过**：描述了具体的现状问题（如"查找文档平均耗时15分钟"）
- **不通过**：只有"当前流程效率不高"等泛泛描述

### A2. 第一性原理
- **判定标准**：一句话本质描述，不含手段
- **通过**：如"让买卖双方建立信任的交易中介"
- **不通过**：包含"系统""平台""功能""模块"等手段词

### A3. 核心特征
- **判定标准**：有 3-4 个维度，包含核心能力和底线
- **通过**：如"实时性、准确性、可追溯性、数据安全"
- **不通过**：只有 1-2 个维度，或缺少安全/质量底线

### A4. 核心目的
- **判定标准**：有 3-5 个，且都是成果表述而非功能描述
- **通过**：如"让用户用最少的操作找到目标内容"
- **不通过**：如"做一个搜索功能"（这是手段）

### A5. 验收标准
- **判定标准**：可衡量，至少覆盖 3 个角色视角
- **通过**：有量化指标或明确判定条件，多角色视角
- **不通过**：只有"用户满意"等无法验证的标准

### A6. 约束摘要
- **判定标准**：精炼，控制在 200 字以内
- **通过**：简洁有力，信息密度高
- **不通过**：超过 200 字或内容空泛

## B. 格式正确性（3 项）

### B1. CLAUDE.md 已生成
- **判定标准**：包含约束摘要 + 下游须知两个章节
- **通过**：两个章节都存在
- **不通过**：缺少任一章节

### B2. CLAUDE.md 约束摘要长度
- **判定标准**：≤ 200 字
- **通过**：字数在限制内
- **不通过**：超过 200 字

### B3. 进度标记
- **判定标准**：项目 index.md 中 L1 标记为 `[x]`
- **通过**：标记正确
- **不通过**：未标记或标记为 `[ ]`

## C. 内容质量（5 项）

### C1. 手段与目的区分
- **判定标准**：产出中无具体功能描述
- **通过**：全篇聚焦"为什么"和"什么叫成功"
- **不通过**：出现"做一个XX""实现XX功能"等手段描述

### C2. 目的粒度适中
- **判定标准**：不太抽象也不太具体
- **通过**：如"让用户用最少的操作找到目标内容"
- **不通过**：太抽象（"提升体验"）或太具体（"做一个带按钮的搜索框"）

### C3. 验收标准可验证
- **判定标准**：有量化指标或明确判定条件
- **通过**：如"新用户 3 步内完成首次操作"
- **不通过**：如"用户感觉更好"

### C4. 核心特征覆盖底线
- **判定标准**：至少包含安全或质量维度
- **通过**：有安全/质量相关特征
- **不通过**：全是业务特征，无底线保护

### C5. 第一性原理纯净
- **判定标准**：不含"系统""平台""功能""模块"等手段词
- **通过**：纯业务本质描述
- **不通过**：混入了技术或实现层面词汇
