# L2↔L3 跨层检查清单（11 项）

## 判定说明

跨层检查使用三级判定：
- **✅ 通过**：符合标准
- **⚠️ 有风险**：存在潜在问题，需要人判断是否可接受
- **❌ 不通过**：明确存在断裂或矛盾

## A. 约束传递（4 项）

### A1. 主体一一对应
- **判定标准**：L3 画像总览的所有主体与 L2 主体链接图一一对应
- **检查方法**：
  1. 从 L2 主体链接图提取所有主体名
  2. 从 L3 画像总览提取所有主体名
  3. 逐个比对
- **✅ 通过**：完全对应
- **❌ 不通过**：L2 有主体在 L3 中缺失（注：若 L3 多出主体，在 C1 处理）

### A2. 痛点事件真实存在
- **判定标准**：L3 痛点列表中关联的 L2 事件确实存在于主体链接图
- **检查方法**：
  1. 从 L3 痛点列表提取所有"L2 事件"引用
  2. 逐条在 L2 主体链接图中验证
- **✅ 通过**：所有引用都存在
- **❌ 不通过**：有痛点引用了 L2 中不存在的事件

### A3. 环境终端一致
- **判定标准**：L3 使用环境的终端类型与 L2 终端分工表一致
- **检查方法**：
  1. 从 L2 终端分工表提取每个主体的终端类型
  2. 从 L3 使用环境提取每个主体的设备类型
  3. 逐个比对
- **✅ 通过**：终端类型匹配
- **❌ 不通过**：L3 描述的终端与 L2 不一致

### A4. 强制回溯引用
- **判定标准**：L2 的强制回溯项（终端分工表）被 L3 正确引用
- **检查方法**：检查 L3 产出中是否引用了 L2 终端分工表的信息
- **✅ 通过**：L3 正确引用了终端分工表
- **❌ 不通过**：L3 未引用或错误引用

## B. 一致性（4 项）

### B1. 行为模式匹配事件
- **判定标准**：L3 画像的行为模式与 L2 主体链接图中该主体参与的事件匹配
- **检查方法**：
  1. 从 L2 提取每个主体参与的事件列表
  2. 从 L3 画像提取行为模式描述
  3. 检查行为模式是否覆盖了主体参与的关键事件
- **✅ 通过**：行为模式与参与事件匹配
- **⚠️ 有风险**：行为模式未完全覆盖，但差距不大
- **❌ 不通过**：行为模式与参与事件明显矛盾

### B2. 设备能力匹配终端
- **判定标准**：L3 使用环境描述的设备能力与 L2 终端分工表中定义的终端能力一致
- **检查方法**：
  1. 从 L2 终端分工表提取每个终端的能力定义
  2. 从 L3 使用环境提取设备能力描述
  3. 检查是否匹配
- **✅ 通过**：设备能力与终端能力一致
- **❌ 不通过**：L3 描述的设备能力超出或低于 L2 定义的终端能力

### B3. 上下文 L2 部分一致
- **判定标准**：L3 产品上下文的 L2 部分与 02-目的边界/index.md 一致
- **检查方法**：对比 L3 上下文中 L2 部分的关键信息与 L2 index.md
- **✅ 通过**：关键信息一致
- **❌ 不通过**：有信息被篡改或遗漏

### B4. 心理特征不矛盾
- **判定标准**：L3 画像的心理特征与 L2 终端分工表中的终端类型不矛盾
- **检查方法**：检查画像心理特征是否与终端使用方式矛盾
- **✅ 通过**：心理特征与终端类型合理
- **❌ 不通过**：如移动端用户被画像为"专注长时间复杂操作"（与移动端特性矛盾）

## C. 断裂检测（3 项）

### C1. 新主体检测
- **判定标准**：L3 是否出现了 L2 未识别的新主体
- **检查方法**：
  1. 从 L3 画像总览提取所有主体
  2. 检查哪些在 L2 主体链接图中未出现
- **✅ 通过**：无新主体
- **❌ 不通过**：有新主体 → 必须回溯 L2 补充主体链接图（最严重的断裂）

### C2. 痛点覆盖度
- **判定标准**：L3 痛点是否覆盖了 L2 所有"纳入"事件的关键环节
- **检查方法**：
  1. 从 L2 边界表提取所有"纳入"事件
  2. 检查每个纳入事件的关键环节是否有对应痛点
- **✅ 通过**：核心纳入事件的关键环节都有痛点
- **⚠️ 有风险**：部分纳入事件无痛点覆盖（可能是非核心环节）
- **❌ 不通过**：大量纳入事件无痛点覆盖

### C3. 终端环境覆盖
- **判定标准**：L2 终端分工表的每个终端在 L3 中都有对应的使用环境描述
- **检查方法**：
  1. 从 L2 终端分工表提取所有终端类型
  2. 检查每个终端在 L3 使用环境中是否有描述
- **✅ 通过**：所有终端都有环境描述
- **❌ 不通过**：有终端在使用环境中未描述
