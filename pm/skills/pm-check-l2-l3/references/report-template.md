# L2↔L3 跨层检查报告模板

```markdown
# L2↔L3 跨层检查报告 — {项目名}

**检查时间**：{时间}
**结论**：✅ 通过 / ⚠️ 有风险 / ❌ 不通过

## 摘要
- 约束传递：{通过数}/4
- 一致性：{通过数}/4
- 断裂检测：{通过数}/3
- **总计：{通过数}/11**
- ⚠️ 有风险项：{列出}
- 🔴 必须修正：{列出 C1 结果}

## 详细判定

### A. 约束传递
- [A1] 主体一一对应：✅ / ❌
  - L2 主体：{列出}
  - L3 画像主体：{列出}
  - L2 中缺失的画像：{若有}
- [A2] 痛点事件真实存在：✅ / ❌
  - L3 引用的 L2 事件：{列出}
  - 不存在的引用：{若有}
- [A3] 环境终端一致：✅ / ❌
  - L2 终端：{列出}
  - L3 环境设备：{列出}
  - 不一致项：{若有}
- [A4] 强制回溯引用：✅ / ❌ {问题描述}

### B. 一致性
- [B1] 行为模式匹配事件：✅ / ⚠️ / ❌ {问题描述}
- [B2] 设备能力匹配终端：✅ / ❌ {问题描述}
- [B3] 上下文 L2 部分一致：✅ / ❌ {问题描述}
- [B4] 心理特征不矛盾：✅ / ❌ {问题描述}

### C. 断裂检测
- [C1] 新主体检测：✅ / ❌
  - 新主体列表：{列出}
  - 处理建议：{必须回溯 L2 补充}
- [C2] 痛点覆盖度：✅ / ⚠️ / ❌
  - 未覆盖的纳入事件：{列出}
- [C3] 终端环境覆盖：✅ / ❌
  - 未覆盖的终端：{列出}

## 严重问题（必须修正）
1. {C1 新主体 或 A3 环境不一致 等严重问题}

## 风险提示（⚠️ 项）
1. {具体风险描述 + 建议处理方式}

## 改进建议（仅 ❌ 项）
1. {具体改进建议}
2. ...
```

## 报告规则

| 规则 | 说明 |
|------|------|
| 结论判定 | 全部 ✅ → 通过；有 ⚠️ 无 ❌ → 有风险；有 ❌ → 不通过 |
| C1 优先级 | C1（新主体）❌ 必须回溯 L2，是最严重的断裂 |
| 三级判定 | ✅ 通过 / ⚠️ 有风险（需人判断） / ❌ 不通过 |
| 终端一致性 | A3/B2 是 L2→L3 衔接最容易出错的环节，重点检查 |
| 风险提示 | ⚠️ 项单独列出，附建议处理方式 |
| 改进建议 | 只针对 ❌ 项，给出可操作的改进方向 |
| 不修改 | 检查报告只输出文本，不写入项目文件 |
