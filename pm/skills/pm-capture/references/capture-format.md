# capture/ 目录格式规范

## 目录结构

### 初始化态（pm-init 创建）

```
capture/
└── index.md             ← 仅标题 "# 项目认知记录"
```

### 完成态（pm-capture 补全后）

```
capture/
├── CLAUDE.md            ← 目录使用规范（框架，不含可变内容）
├── index.md             ← 总索引（顶部高影响区 + 按类型索引）
├── decisions.md         ← 全部决策记录
├── insights.md          ← 全部洞察记录
└── frameworks.md        ← 全部框架认知记录
```

## CLAUDE.md 格式（使用规范）

```markdown
# capture/ 目录规范

## 定位
项目认知记录——从 SKILL 执行对话中提取的决策、洞察、框架认知。

## 文件结构
- index.md: 总索引（顶部高影响区 + 按类型索引），扫描入口
- decisions.md: 决策记录（D 前缀），详情
- insights.md: 洞察记录（I 前缀），详情
- frameworks.md: 框架认知（F 前缀），详情

## 记录类型
- decision: 选择（X vs Y），可被推翻（状态：active/superseded/revoked）
- insight: 对产品或行业的认知
- framework: 心智模型，对产品本质的理解

## 使用规则
- 写入：pm-capture SKILL 独占写入权
- 读取：任何层 SKILL 可读取
- index.md 是扫描入口，详细内容在类型文件中（引用不复制）
- 每条记录至少 1 个标签
- 高影响条目在 index.md 顶部集中展示
```

## index.md 格式

```markdown
# 项目认知记录

## 高影响认知（跨层必看）
- D001: 标题 `标签1` `标签2`
- F001: 标题 `标签1`
- I001: 标题 `标签1`

## 框架认知
- F001: 标题 — 一句话概括 `标签1` `标签2`

## 洞察
- I001: 标题 — 一句话概括 `标签1`

## 决策
- D001: 标题 — 一句话概括 `标签1` `标签2`
```

### index.md 编写规则

| 规则 | 说明 |
|------|------|
| 引用不复制 | 只放 ID + 标题 + 标签，详细内容在类型文件中 |
| 高影响区 | 标记了 `影响级别: high` 的条目集中到顶部 |
| 按类型分组 | 框架认知 → 洞察 → 决策 |
| 标签用反引号 | 便于 grep 搜索 |

## 三种记录格式

### 决策（decisions.md）

```markdown
### D001: 标题
- **标签**: tag1, tag2
- **来源**: L2
- **状态**: active
- **影响级别**: high
- **内容**: 具体描述
- **原因**: 为什么做这个选择
- **影响**: 影响哪些下游工作
```

| 字段 | 规则 |
|------|------|
| 标签 | 至少 1 个，逗号分隔 |
| 来源 | L1/L2/L3/L4/L5，标注在哪一层的对话中产生 |
| 状态 | active（生效）/ superseded（被新决策替代）/ revoked（已废除） |
| 影响级别 | high（高影响）/ 留空（普通） |
| 原因 | 必填，说明为什么做这个选择 |
| 影响 | 选填，说明影响哪些下游工作 |

### 洞察（insights.md）

```markdown
### I001: 标题
- **标签**: tag1, tag2
- **来源**: L2
- **影响级别**: high
- **内容**: 具体描述
- **上下文**: 这个认知从哪来的
- **影响**: 对设计的影响
```

| 字段 | 规则 |
|------|------|
| 上下文 | 必填，说明这个认知的产生背景 |
| 影响 | 选填，说明对设计有什么影响 |

### 框架认知（frameworks.md）

```markdown
### F001: 标题
- **标签**: tag1, tag2
- **来源**: L1
- **影响级别**: high
- **内容**: 具体描述
- **推理过程**: 怎么得出这个认知的
- **影响**: 对整体设计的影响
```

| 字段 | 规则 |
|------|------|
| 推理过程 | 必填，说明从什么信息推导出这个认知 |
| 影响 | 必填，框架认知通常影响整体方向 |

## ID 编号规则

- D 前缀：决策（Decision）
- I 前缀：洞察（Insight）
- F 前缀：框架认知（Framework）
- 三类独立编号：D001/D002...、I001/I002...、F001/F002...
- 新记录编号 = 当前最大编号 + 1
- 编号一旦分配不可重用（即使状态变为 revoked）
