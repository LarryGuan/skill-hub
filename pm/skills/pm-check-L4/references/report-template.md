## 子单元检查报告

```markdown
# L4 检查报告（子单元：{终端名}） — {项目名}

**检查时间**：{时间}
**检查模式**：子单元检查（{终端名}）
**结论**：✅ 通过 / ❌ 不通过

## 摘要
- 完整性：{通过数}/4
- 质量：{通过数}/4
- 回溯：{通过数}/2
- **总计：{通过数}/10**

## 详细判定

### E. 完整性
- [E1] 关系事件覆盖：✅ / ❌ {问题描述}
- [E2] 主体覆盖：✅ / ❌ {问题描述}
- [E3] 场景要素完整：✅ / ❌ {问题描述}
- [E4] 目录结构规范：✅ / ❌ {问题描述}

### F. 质量
- [F1] 场景回答"在什么情况下做什么"：✅ / ❌ {问题描述}
- [F2] 设备环境嵌入：✅ / ❌ {问题描述}
- [F3] 参与者匹配：✅ / ❌ {问题描述}
- [F4] 场景粒度适中：✅ / ❌ {问题描述}

### G. 回溯
- [G1] L2 终端场景清单覆盖：✅ / ❌ {问题描述}
- [G2] L3 使用环境一致：✅ / ❌ {问题描述}

## 改进建议（仅不通过项）
1. {具体改进建议}
2. ...
```

---

## 报告规则

| 规则 | 说明 |
|------|------|
| 结论判定（整体） | 全部 24 项 ✅ → 通过；任何 1 项 ❌ → 不通过 |
| 结论判定（子单元） | 全部 10 项 ✅ → 通过；任何 1 项 ❌ → 不通过 |
| 问题描述 | ❌ 必须附上具体位置和不符合的原因 |
| 改进建议 | 只针对不通过项，给出可操作的改进方向 |
| 不修改 | 检查报告只输出文本，不写入项目文件 |
| 强制修正（整体） | A1（关系事件覆盖）和 D2（痛点覆盖）不通过必须修正 |
| 子单元独立性 | 子单元检查不通过不影响其他子单元的检查 |
| 子单元非替代 | 子单元检查通过不代表整体通过，整体组装后还需交叉分析 |
